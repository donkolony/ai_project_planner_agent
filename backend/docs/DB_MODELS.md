# Database Models Module

## Overview

The `db_models.py` module defines the database schema for the AI Project Planner Agent backend using SQLModel. It specifies the persistent data structures stored in the database, including table definitions, field types, and relationships. This module serves as the foundation for data persistence and retrieval operations.

## Architecture

The module uses SQLModel's declarative approach to define database tables:

- Inherits from `SQLModel` base class
- Uses `Field` for column definitions
- Supports both SQLAlchemy and Pydantic features
- Enables type-safe database operations

## PlanDB Model

### Class Definition
```python
class PlanDB(SQLModel, table=True):
    # Database table definition
```

### Table Structure

#### Primary Key
- `id: str` - Unique identifier (UUID)
  - Generated using `uuid.uuid4()`
  - Primary key constraint
  - Indexed for fast lookups

#### Project Information
- `project_name: str` - User-provided project name
  - Required field
  - Stores the project title

- `description: str` - Detailed project description
  - Required field
  - Contains user input about project goals

#### Technology Stack
- `tech_stack: str` - Technology preferences
  - Required field
  - JSON-encoded string of technology list
  - Allows flexible technology specification

#### AI-Generated Content
- `recommended_tech_stack: str` - AI-suggested technologies
  - Optional field (defaults to "[]")
  - JSON-encoded string
  - Stores AI recommendations

- `summary: str` - AI-generated project overview
  - Required field
  - High-level architectural summary
  - Generated by AI service

- `phases: str` - Project development phases
  - Required field
  - JSON-encoded string of phase data
  - Contains structured task breakdowns

## Data Types and Constraints

### String Fields
- All text fields use SQLAlchemy `String` type
- No length limits specified (database-dependent)
- UTF-8 encoding support

### JSON Storage
Complex data structures stored as JSON strings:
- Technology stacks: `["Python", "FastAPI", "SQLModel"]`
- Phase structures: `[{"name": "Setup", "tasks": ["Task 1", "Task 2"]}]`
- Enables flexible schema evolution

### UUID Primary Key
- Universally unique identifiers
- Collision-resistant
- URL-safe for API endpoints
- Efficient indexing

## Database Schema Generation

### Table Creation
The model automatically generates database tables:
```sql
CREATE TABLE plandb (
    id VARCHAR PRIMARY KEY,
    project_name VARCHAR NOT NULL,
    description VARCHAR NOT NULL,
    tech_stack VARCHAR NOT NULL,
    recommended_tech_stack VARCHAR DEFAULT '[]',
    summary VARCHAR NOT NULL,
    phases VARCHAR NOT NULL
);
```

### Metadata Integration
- Registered with `SQLModel.metadata`
- Included in `create_all()` operations
- Supports database migration tools

## CRUD Operations

### Create
New plans instantiated and added to sessions:
```python
plan = PlanDB(
    project_name="Project Name",
    description="Description",
    # ... other fields
)
session.add(plan)
session.commit()
```

### Read
Plans retrieved by ID or queried in bulk:
```python
plan = session.get(PlanDB, plan_id)
plans = session.query(PlanDB).all()
```

### Update
Fields modified and changes committed:
```python
plan.summary = "Updated summary"
session.commit()
```

### Delete
Plans removed from database:
```python
session.delete(plan)
session.commit()
```
